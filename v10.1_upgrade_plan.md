# v10.1 升级计划：内容设计 + 期刊自适应 + 图表修复

> 制定时间: 2026-05-30
> 核心理念: 指定一个期刊 → 从该期刊的论文中学习风格 → 按该期刊的期望写论文
> **不是** 写死 IEEE TCSVT 风格，**而是** 让系统能学习任意期刊

---

## 一、核心问题诊断

### 问题 A：风格系统"形似神不似"

现状有 4 层风格体系（P2/P4/Venue Profile/Ref PDF），但实际注入到 LLM prompt 的风格指导极其薄弱：

```
# 每章的 _build_style_instruction() 实际只注入了：
- 学术用词列表（从 ref_pdf 提取的 15-20 个词）
- 关键句式模板（3-5 个）
- 3-5 条硬编码的"重要要求"
- venue adapter 的一个写作风格 dict（tone, argument_depth 等，只作为一句话描述）
```

**缺失的关键维度**：
- ❌ 期刊特有的内容编排逻辑（TCSVT 的 Methodology 先系统总览再模块细节 vs TIP 的先问题建模再方法设计）
- ❌ 期刊的内容侧重点（TCSVT 偏系统实现效率 vs TPAMI 偏理论完备性）
- ❌ 期刊的图片风格偏好（TCSVT 偏系统框图 vs CVPR 偏可视化结果对比）
- ❌ 期刊的论证节奏（哪里详写、哪里略写、哪里放数据、哪里放图表）
- ❌ 从 ref_pdf 学到的风格只取了"用词"和"句式"，丢失了"内容组织"和"论证节奏"

### 问题 B：内容设计缺乏策略性

现状的写法是"每章机械执行"：
1. Phase 0.6-0.9 做了动机线程、范例学习、写作矩阵等规划
2. 但这些规划的产出在 Phase 1-5 的 prompt 中**利用率极低**
3. 结果是每章各写各的，缺乏"这篇论文应该重点讲什么"的策略性决策

**具体表现**：
- ❌ 没有基于创新点做"重点分配"（创新点A重点在3.2节，创新点B重点在4.3节）
- ❌ 没有基于目标期刊调整"深度梯度"（TPAMI的推导要完整 vs CVPR的推导可简化）
- ❌ 动机线程只生成了锚点文本，没有验证章节是否实际遵循了这些锚点
- ❌ 内容选择（写什么/不写什么）完全靠 LLM 即兴发挥，缺乏事前规划

### 问题 C：图表管线 fig4/5 失败

这已在之前分析过，是独立的技术问题（路由、JSON解析、评分），在此一并修复。

## 二、v10.1 三大改动方向

### 方向 1：期刊风格自适应系统（Journal Style Learning）

**目标**：给定任意期刊名称（如 "IEEE TIP"），系统能从该期刊的已发表论文中自动学习并应用该期刊特有的写作风格。

**不碰**：现有的 VenueProfile 参数化配置（页数、字数、消融数量等硬参数保留），只增强**软约束**维度。

#### 1.1 新建 `JournalStyleLearner` — 从 ref_pdf 深度学习期刊风格

位置：`agent/journal_style_learner.py`（新建）

当前 `ref_pdf_analyzer.py` 的分析维度太浅（只提取了用词、句式、段落组织）。需要增加：

| 分析维度 | 提取什么 | 注入到哪里 |
|----------|----------|-----------|
| **内容编排模式** | 每章的子节组织顺序、逻辑推进模式 | `structure_planner.py` 的 Content Checklist |
| **论证节奏** | 哪里放数据、哪里放图表、哪里放引用 | 每章的 prompt 的 content_strategy 段 |
| **深度梯度** | 每个子节的"推导深度"（完整/概述/省略） | venue_adapter 的 section_prompt_hint |
| **内容侧重点** | 该期刊的 Methodology 更偏理论/系统/实验 | `_build_style_instruction()` |
| **图片风格偏好** | 架构图风格（简洁线条/详细标注）、结果图风格 | figure_generator 的设计引导 |
| **审稿偏好** | 从论文的 rejection/revision 痕迹推断 | 质量门控的评审标准 |

实现方式：
```python
class JournalStyleLearner:
    """从 ref_pdf 深度学习目标期刊的写作风格"""
    
    def learn(self, papers: List[Paper], venue_name: str) -> JournalStyleProfile:
        """
        输入：ref_pdf 中的论文列表 + 目标期刊名
        输出：JournalStyleProfile（结构化的风格画像）
        
        分析流程：
        1. 用 LLM 分析每篇论文的 6 层特征（复用 exemplar_learner 的框架）
        2. 增加 4 个新维度：content_pattern, argument_rhythm, depth_gradient, figure_preference
        3. 聚合多篇论文得到期刊级风格画像
        4. 与已有 venue_profile 合并（期刊特有规则覆盖通用规则）
        """
```

#### 1.2 扩展 `VenueProfile` 基类 — 增加软约束字段

位置：`config/venue_profiles/base_profile.py`（修改）

新增字段（全都有默认值，不破坏现有 11 个 profile）：

```python
# ---- 内容编排软约束（从 ref_pdf 学习填充）----
content_patterns: Dict[str, Any] = field(default_factory=dict)
# 例: {"Introduction": {"opening": "problem_motivation", "contributions": "bulleted_3items", ...}}

argument_rhythm: Dict[str, Any] = field(default_factory=dict)
# 例: {"Methodology": {"theory_first": True, "derivation_depth": "full", ...}}

depth_gradients: Dict[str, str] = field(default_factory=dict)
# 例: {"Methodology.3.1": "full_derivation", "Methodology.3.2": "overview"}

figure_preferences: Dict[str, Any] = field(default_factory=dict)
# 例: {"architecture": {"style": "clean_lines", "detail_level": "moderate"}}

# ---- 期刊特有的内容侧重点 ----
content_emphasis: Dict[str, str] = field(default_factory=dict)
# 例: {"methodology": "system_implementation", "experiments": "cross_dataset"}

reviewer_preferences: List[str] = field(default_factory=list)
# 例: ["always include complexity analysis", "prefer real-world datasets over synthetic"]
```

#### 1.3 升级 `_build_style_instruction()` — 所有章节通用

位置：`agent/skill_orchestrators/` 下所有 chN 文件（统一重构）

当前每个章节的 `_build_style_instruction()` 各自硬编码了"特殊要求"。改为统一从一个中心化的风格管理器获取：

```python
# 新建 agent/style_manager.py
class StyleManager:
    """统一的风格管理中心"""
    
    def __init__(self, venue_adapter, journal_style_profile):
        self.venue = venue_adapter
        self.journal_style = journal_style_profile
    
    def build_chapter_style_instruction(self, chapter_name: str) -> str:
        """
        为指定章节构建完整的风格指导，包含：
        1. 期刊级内容编排模式（从 journal_style_profile 获取）
        2. 期刊级论证节奏和深度梯度
        3. 期刊级内容侧重点
        4. 通用学术写作规范（从 style_guide.md 获取）
        5. 期刊特有的 Red Flags（从 ieee_trans_style_profile 或学习得到）
        """
```

### 方向 2：智能内容编排系统（Content Strategy）

**目标**：在开始写每章之前，先做"写什么、不写什么、重点写什么"的策略性规划，而不是让 LLM 即兴发挥。

#### 2.1 新建 `ContentStrategist` — 内容策略规划器

位置：`agent/content_strategist.py`（新建）

```python
class ContentStrategist:
    """基于创新点和目标期刊，规划每章的内容策略"""
    
    def plan(self, project_data, innovation_points, venue_adapter, 
             journal_style, motivation_thread) -> ContentStrategy:
        """
        输出 ContentStrategy：
        {
            "Introduction": {
                "focus_areas": ["问题的重要性", "现有方法的3个关键局限"],
                "depth_map": {"背景": "brief", "局限分析": "deep", "贡献": "moderate"},
                "innovation_allocation": {"创新点1": "主导", "创新点2": "提及"},
                "must_include": ["至少1个具体数据支撑问题严重性"],
                "should_avoid": ["不要在此节详细描述方法细节"],
                "figure_plan": [{"type": "teaser", "content_hint": "..."}],
                "target_emphasis": "问题驱动，gap 明确"
            },
            "Methodology": {
                "subsection_plan": [
                    {"name": "3.1 Overview", "depth": "overview", "innovation": "none"},
                    {"name": "3.2 Module A", "depth": "full_derivation", "innovation": "创新点1"},
                    {"name": "3.3 Module B", "depth": "moderate", "innovation": "创新点2"},
                    {"name": "3.4 Training", "depth": "brief", "innovation": "none"},
                ],
                ...
            }
        }
        """
```

关键特性：
- **创新点分配矩阵**：每个创新点对应到重点阐述的章节位置
- **深度梯度**：基于期刊偏好和内容重要性，决定每个子节的深度
- **must_include / should_avoid**：明确告诉 LLM 写什么和不写什么
- **与动机线程联动**：验证每个章节是否覆盖了动机锚点

#### 2.2 升级 `structure_planner.py` — 增加内容策略维度

位置：`agent/skill_orchestrators/structure_planner.py`（修改）

当前的 Content Checklist 是静态的 5-7 项。改为：
- 接收 ContentStrategist 的策略输出
- 每章的 Content Checklist 变为"必须覆盖的内容 + 建议覆盖的内容 + 建议跳过的内容"
- 增加"内容预算"维度（该期刊的 Methodology 分配多少字给理论推导 vs 系统描述 vs 实验设计）

#### 2.3 升级章节生成 prompt — 注入内容策略

每章的 prompt 从当前的"你是一个论文写作专家，请写 XX 节"变为：

```
你正在为 {venue_name} 撰写论文。

**本节内容策略**：
- 目标：{content_strategy['focus_areas']}
- 深度梯度：{content_strategy['depth_map']}
- 创新点分配：{content_strategy['innovation_allocation']}
- 必须覆盖：{content_strategy['must_include']}
- 应当避免：{content_strategy['should_avoid']}

**期刊风格参考**：
- 该期刊的 {chapter_name} 通常的组织方式：{journal_style['content_pattern']}
- 该期刊偏好 {journal_style['argument_rhythm']}

**动机线程锚点**：
- {motivation_thread[chapter_name]}

{原有的 innovation_points, experiment_design 等项目数据}
```

#### 2.4 新增"叙事弧线验证"步骤

在每章生成后，验证内容是否遵循了内容策略：

```python
def verify_content_strategy(chapter_content, content_strategy, motivation_thread):
    """
    验证：
    1. must_include 的内容是否实际覆盖
    2. should_avoid 的内容是否误入
    3. 动机锚点是否体现
    4. 深度梯度是否合理（该详的是否详了，该略的是否略了）
    """
```

这作为一个轻量级验证步骤，不增加 LLM 成本（关键词匹配 + 规则检查），在质量门控之前执行。

## 五、核心设计原则（不变）

| 角色 | 实现 | 说明 |
|------|------|------|
| **裁判** | 代码 | 管线阶段路由、编译验证、内容策略验证、质量门控 |
| **规则书** | MD引导文件 | 设计指南、评审标准、期刊风格模板 |
| **运动员** | LLM | 内容创作、风格学习、策略规划、图表设计、评审打分 |

新增原则：
- **期刊风格 = 从论文学习，不从代码硬编码** — base_profile 的软约束字段默认为空，由 JournalStyleLearner 从 ref_pdf 填充
- **内容策略 = 先规划后执行** — Phase 0.65 的 ContentStrategy 输出驱动 Phase 1-5 的内容生成
- **不绑定任何具体期刊** — 系统能适配任意期刊，TCSVT 只作为第一个完整填充的示例

## 六、预期效果

| 指标 | v10 现状 | v10.1 目标 |
|------|----------|-----------|
| 期刊风格适配 | 静态参数（页数/字数/消融数），无内容编排学习 | 从 ref_pdf 动态学习内容编排、论证节奏、深度梯度 |

| 风格注入 | 用词列表 + 3-5 条硬编码要求 | 完整的期刊风格画像（内容编排 + 论证节奏 + 图片偏好 + 审稿偏好） |
| 图表 fig4/5 | FAIL（占位符） | LLM TikZ 降级，不再直接占位 |
| 评审 JSON | 脆弱（单正则） | 多层回退，健壮解析 |
| 换期刊成本 | 需改 5+ 个文件的硬编码 | 只需在 project_config.py 改 TARGET_VENUE + 放入新期刊的 ref_pdf |
