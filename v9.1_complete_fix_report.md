## 详细修复说明

### Fix A1: `####` 标题残留 (CRITICAL)

**文件**: `tools/latex_converter.py`

**根本原因**: 
- 步骤 14 的 `#` → `\#` 转义在步骤 8 的 Markdown 标题转换之后执行
- `#### 3.7.1` 先被转义成 `\#\#\#\# 3.7.1`，导致标题正则无法匹配

**修复方案**:
1. 扩展标题正则从 `#{1,3}` 到 `#{1,6}`，支持所有 Markdown 标题级别
2. 添加 `####` → `\paragraph{}`，`#####` → `\subparagraph{}`，`######` → `\subparagraph{}`
3. 修改 `#` 转义逻辑，只转义非标题位置的 `#`
4. 添加 Unicode 清理：`latex.replace('\ufffd', '')`

**验证**: 40 → 0 处 ✅

---

### Fix A2: 表格宽度溢出 (CRITICAL)

**文件**: `tools/latex_converter.py`，函数 `replace_table()`

**根本原因**: `col_spec = 'l' * col_count` 没有宽度控制

**修复方案**:
1. 列数 > 5：`table*` + `p{0.13\textwidth}` + `tabular*`
2. 列数 > 3：`tabular*` + `\linewidth`
3. 列数 ≤ 3：标准 `tabular`

**验证**: 7 列 `lllllll` → `tabular*` + `p{}` ✅

---

### Fix A3: 星号残留 (HIGH)

**根本原因**: `table*` 和 `tabular*` 中的 `*` 被步骤 15 的 `*` → `\textit{}` 转换影响

**修复方案**:
1. 在步骤 11 表格转换后立即保护新生成的 `table*` 和 `tabular*`
2. 在步骤 15 的所有星号转换之后恢复

**验证**: 1 → 0 处 ✅
