### 2.2 硬约束系统

#### 创建的文件

**`agent/academic_style_checker.py`** (约 400 行)

功能模块：
1. **AI 风格词汇检测** (`_check_ai_words`)
   - 检测 28 个 AI 风格词汇
   - 提供替代表达建议
   - 返回位置和上下文

2. **括号长度检查** (`_check_parenthesis_length`)
   - 检测括号内超过 20 词的内容
   - 建议拆分为独立句子

3. **破折号滥用检测** (`_check_dash_usage`)
   - 统计破折号使用次数
   - 建议改用逗号或括号

4. **句子长度检查** (`_check_sentence_length`)
   - 计算平均句长
   - 检测超过 40/50 词的长句
   - 提供长度分布统计

5. **段落结构检查** (`_check_paragraph_structure`)
   - 检测过短（<3 句）或过长（>8 句）段落
   - 建议拆分或合并

6. **时态一致性检查** (`_check_tense_consistency`)
   - 按章节检查时态规范
   - Related Work → 过去时
   - Methodology → 现在时
   - Experiments → 过去时（结果）+ 现在时（图表）

#### 评分机制

```python
def _calculate_score(self, issues, word_count):
    score = 100.0
    
    for issue in issues:
        if issue["type"] == "ai_flavored_word":
            score -= 3  # 每个 AI 词汇扣 3 分
        elif issue["type"] == "long_parenthesis":
            score -= 2  # 每个长括号扣 2 分
        elif issue["type"] == "tense_inconsistency":
            score -= 10  # 时态不一致扣 10 分
        # ...
    
    return max(0.0, min(100.0, score))
```

**通过阈值**: 75 分  
**强制不通过**: 存在 critical 级别问题

# v9.2: 学术风格二次评价（硬约束机制）
from agent.academic_style_checker import AcademicStyleChecker

llm_score = sum(report.dimensions.values()) / len(report.dimensions)
# 风格评分权重 30%，LLM 评分权重 70%
report.overall_score = llm_score * 0.7 + style_score * 0.3

# 通过条件
report.passed = (
    report.overall_score >= self.PASS_THRESHOLD

## 4. 架构对比

### v9.1（硬编码方式）

```
AI 生成内容 → latex_converter.py (硬编码替换) → LaTeX 输出
                    ↓
              9 个词汇替换
              长括号转换
```

**问题**:
- 仅在最后阶段处理，AI 生成时没有引导
- 硬编码规则难以维护
- 无法检测复杂模式（如时态、句子长度）

### v9.2（柔性引导 + 硬约束）

```
风格指南注入 → AI 生成内容 → QualityGate (风格检查) → 通过？
     ↓              ↓              ↓
 柔性引导      学术化写作      28 个词汇检测
                           括号/句长/时态检查
                           评分融合（30% 权重）
                                ↓
                           不通过 → 自动重试
```

**优势**:
- AI 从源头学术化写作（柔性引导）
- 二次评价确保合规（硬约束）
- 可扩展性强（添加新词汇/规则只需修改 checker）
- 评分融合机制更科学

## 7. 关键设计决策

### 为什么风格评分权重 30%？
- LLM 评估（70%）: 关注内容质量、逻辑连贯、引用自然
- 风格检查（30%）: 关注词汇选择、句式规范、时态一致性
- 平衡点：风格重要但不应该压倒内容质量

### 为什么移除硬编码而不是保留？
- 硬编码导致维护困难（每次添加新词汇需要修改 converter）
- 风格检查器更灵活（配置化、可扩展）
- 柔性引导从源头解决问题，硬编码只是后处理

### 为什么风格指南只截取 2000 字？
- Prompt 长度有限制
- 截取最关键部分（禁用词汇、句式规范）
- 完整指南可通过 skill 文件引用
