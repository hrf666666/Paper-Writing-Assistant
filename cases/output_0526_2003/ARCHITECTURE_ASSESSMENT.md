## 六、下一代迭代路线图

### v8.0 核心改进（优先级排序）

#### P0: 关键修复（必须）
1. **参考文献系统重构**
   - 当前问题：Semantic Scholar API 限流导致27个[?]未解析
   - 方案：本地引文库 + 多源验证(S2 + CrossRef + Google Scholar)
   - 预期效果：引用解析率从 3.6%(1/28) 提升到 95%+

2. **Abstract-Experiments 数据一致性**
   - 当前问题：Abstract 提到的数值与 Experiments 不匹配
   - 方案：Abstract 生成时注入 Experiments 的实际数据表，强制引用
   - 预期效果：消除数值不一致

3. **去重逻辑**
   - 当前问题：Discussion/Limitations 在论文中重复出现
   - 方案：`_run_output_phase()` 中增加去重检查

#### P1: 架构增强（借鉴 auto_research_agent）
4. **SQLite 持久化记忆**
   - 借鉴 auto_research_agent 的三层记忆架构
   - 替换当前 JSON 文件，支持无限历史和复杂查询
   - 增加 Pareto 前沿追踪和跨实验因果分析

5. **ToolTrace 反欺骗机制**
   - 关键事实（引用作者、发表年份、期刊）从工具结果提取，不信任 LLM 叙述
   - 特别适用于参考文献验证和创新点事实核查

6. **两级 LLM Failover**
   - Level 1: 同 Provider 内模型链降级
   - Level 2: 跨 Provider 自动切换
   - 增加 Provider 健康追踪和冷却机制

#### P2: 写作质量提升
7. **物理模型严谨化引擎**
   - 检测公式中的"形式正确但内涵空洞"表达
   - 为每个公式要求：首次定义 + 物理意义 + 适用条件
   - 自动检测符号一致性

8. **深度消融实验设计**
   - 当前消融仅4组，TCSVT标准需8+组
   - 自动生成消融矩阵（每个模块 on/off + 两两组合）
   - 增加 BadPix 指标（δ<0.03/0.07/0.1）为标配

9. **失败案例自动分析**
   - 在 Experiments 章节自动增加"Failure Analysis"小节
   - 选择 worst-case 场景进行深度分析
   - 生成误差热力图和 EPI 线可视化

#### P3: 用户体验优化
10. **实时进度 Dashboard**
    - WebSocket 推送各阶段进度
    - 可视化质量评分变化曲线
    - 支持运行中实时修改参数

11. **多论文批量生成**
    - 支持一个项目生成多个版本（不同模型/不同风格）
    - 自动对比选优

12. **LaTeX 编译集成**
    - 安装 TeX Live，支持自动编译验证
    - TikZ 架构图渲染验证
    - 公式渲染正确性检查
