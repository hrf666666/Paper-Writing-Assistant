## 2. 核心改进效果

### 2.1 公式转换修复 ✅ 巨大改进

**根因**: `convert_display_math` 将所有 `__ORIGMATH_` 标记（包括行内公式）错误地包装为 `equation` 环境。

**修复**: 合并为单一 `convert_math_marker` 函数，按 `math_type` 区分行内/显示：
- `inline` → `$...$`
- `display` → `equation` 或 `align` 环境

**效果**:
| 指标 | v2.0 | v3.0 |
|------|------|------|
| 公式数 | 286 | **15** |
| `\cr` 错误 | 52 | **0** |
| `\eqno` 错误 | 4 | **1** |
| `\spacefactor` 错误 | 8 | **0** |

**分析**: 公式数量从 286 降至 15 (-94.8%)，消除几乎所有 `\cr` 和 `\spacefactor` 错误。公式转换逻辑修复是本轮最关键的改进。

### 2.2 编译日志去重 ✅ 巨大改进

**根因**: 一个未闭合的 `$` 会级联产生 51 个 `Missing $ inserted` 错误。

**修复**: 添加 `CASCADE_SECONDARY` 分类规则和 `_deduplicate_cascade_errors` 方法，每类级联错误只保留前 N 个代表样本。

**效果**:
| 指标 | v2.0 | v3.0 |
|------|------|------|
| 原始错误 | 193 | 102 |
| 去重后 | 193 (无去重) | **22** |
| 去重率 | 0% | **78.4%** |

### 2.3 Checkpoint 排序修复 ✅ 完全修复

**根因**: `phase_index` 从 JSON 恢复后为字符串 `"0_5"`，无法与整数比较排序。

**修复**: 
1. `from_dict` 中强制 `int(d['phase_index'])`
2. `get_summary` 中使用 `_safe_phase_index` 防御性转换

**效果**: Pipeline 完成后正常输出检查点摘要，无 TypeError 崩溃。

### 2.4 PyMuPDF 安装 ✅ 高级验证可用

**效果**: PDF 结构验证不再降级为基础检查。首次获得真实数据：
- 页数: 15 页 ✅
- 空白页: [13, 14, 15] ⚠️
- 文本覆盖率: 9.83% ❌ (目标 >30%)

## 3. 仍然存在的问题

### 3.1 PDF 后 3 页空白

**现象**: 第 13-15 页为空白页，文本覆盖率仅 9.83%。

**可能原因**:
1. LaTeX 浮动体（figure/table）位置不当导致内容前移
2. 公式语法错误导致编译器跳过内容
3. 内容不足 15 页，LaTeX 填充空白页

### 3.2 Missing $ 级联错误 (51 个 → 去重后 3 个)

**分析**: 去重后仍显示 3 个 `Missing $ inserted` 错误，说明仍有公式定界符问题。

**可能的根因**:
1. LLM 生成的行内公式中包含未配对的 `$`
2. `\textbf{...}` 或 `\textit{...}` 在数学模式中使用不当
3. 公式中有 `&` 字符未在表格/align 环境中

### 3.3 Display math should end with $$

**分析**: 仍有 1 个显示公式未正确闭合，可能是跨行 `$$...$$` 配对问题。

### 3.4 Extra \endgroup (3 个)

**分析**: 可能与表格环境或公式环境的嵌套不匹配有关。

### 3.5 结构分析修复未触发

**日志**: `[Phase 8.5] 无问题可修复，停止重试`

**原因**: 结构分析 (`_analyze_tex_structure`) 未检测到问题，因为：
1. 去重后的 22 个问题中，环境嵌套问题可能不在结构分析检测范围内
2. 当前错误以 `Missing $` 为主，而非 `\begin/\end` 不匹配

### 3.6 L3 学术质量评分为 0

**现象**: Phase 9 评价 L3=0, Grade=D

**原因**: 可能是 LLM 评价器的问题，或 `output_evaluator.py` 评分逻辑变更。
